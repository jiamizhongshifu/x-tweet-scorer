# Phoenix Mapping · 14 动作 → 5 维度完整映射

> 本文档说明本 Skill 的 5 个评分维度如何聚合自 Phoenix 源码中的 14 个原生动作概率。
> 引用源码路径：`xai-org/x-algorithm/phoenix/` 和 `home-mixer/` 中的 Scorer 实现。

## 源码中的 14 个动作（原文照录）

```
Predictions:
├── P(favorite)        点赞
├── P(reply)           回复
├── P(repost)          转推（无评论）
├── P(quote)           引用转推（带评论）
├── P(click)           点击链接/卡片
├── P(profile_click)   点击作者头像/名字进入主页
├── P(video_view)      视频被观看
├── P(photo_expand)    图片被展开
├── P(share)           外部分享（复制链接/分享到其他平台）
├── P(dwell)           停留时长（被读完）
├── P(follow_author)   关注作者
├── P(not_interested)  点击"不感兴趣"
├── P(block_author)    拉黑作者
├── P(mute_author)     静音作者
└── P(report)          举报
```

源码：参考 `home-mixer` 中 Weighted Scorer 的实现。

## 聚合理由：为什么是这 5 个维度而不是 14 个

直接暴露 14 个数字会让用户陷入"看不懂哪个最重要"的认知过载。聚合后的 5 个维度按"创作者可控的优化方向"对齐：

- 🔥 **分发** vs 💬 **对话**：都是"主动行为"，但分发是把内容推向陌生人（出圈），对话是和现有受众深聊（社区），优化策略不同。
- 👀 **沉浸**：捕捉被动信号（dwell + 各种点击/展开），这是算法层面的"暗物质"，独立成维。
- 🎯 **关注**：把"对内容的反应"和"对作者的反应"分开，因为后者是涨粉转化的核心。
- ⚠️ **风险**：所有负向动作合并，因为它们在加权公式里都是减法，且任何一个触发都会拖垮总分。

## 维度详细映射

### A · 分发力 (Amplification) — 权重 0.35

| 原生动作 | 在维度内的子权重 | 说明 |
|---|---|---|
| P(repost) | 0.50 | 纯转发：单向分发，对算法最有价值（最强出圈信号） |
| P(quote)  | 0.30 | 引用：带评论的二次创作，分发 + 对话双信号 |
| P(share)  | 0.20 | 外部分享：跨平台扩散，长尾价值 |

**核心可优化要素**：
- Hook 强度（首句 7 秒能否抓住人）
- "可引用句"（能不能截图发朋友圈的金句）
- 身份标签（转发它能强化转发者的什么人设）

**对应源码组件**：`OON Scorer`（出圈逻辑）、`Phoenix Retrieval`（Two-Tower 匹配陌生人）

---

### C · 对话力 (Conversation) — 权重 0.25

| 原生动作 | 在维度内的子权重 | 说明 |
|---|---|---|
| P(reply)    | 0.70 | 回复：算法极度看重（对话延长 session，产生新内容） |
| P(favorite) | 0.30 | 点赞：最轻量的正向反馈 |

**核心可优化要素**：
- 结尾留可回答的钩子（不是"你怎么看"这种泛问）
- 轻度争议性（触发"我必须纠正你"的反应）
- 情感共鸣点

**对应源码组件**：`Weighted Scorer`（reply 权重通常最高之一）

---

### T · 沉浸力 (Attention) — 权重 0.20

| 原生动作 | 在维度内的子权重 | 说明 |
|---|---|---|
| P(dwell)        | 0.50 | 停留时长：**最被低估的信号**，所有人都会贡献 |
| P(click)        | 0.20 | 点击链接 |
| P(video_view)   | 0.15 | 视频观看 |
| P(photo_expand) | 0.15 | 图片展开 |

**核心可优化要素**：
- 信息密度（不能水）
- 结构化（数字列表、对比、悬念）
- 必要的长度（反直觉：算法不一定喜欢短推）

**对应源码组件**：`Phoenix Ranking Transformer`（dwell 是它预测的独立目标之一）

---

### F · 关注力 (Authority) — 权重 0.10

| 原生动作 | 在维度内的子权重 | 说明 |
|---|---|---|
| P(profile_click)  | 0.40 | 点击主页：好奇但还没决定关注 |
| P(follow_author)  | 0.60 | 关注：最终的转化 |

**核心可优化要素**：
- 作者身份感（这条推文展现了你"是个什么人"）
- 持续阅读价值的暗示（关注你能持续看到什么）
- 专业度信号

**对应源码组件**：`User Tower`（用户 embedding 包含关注关系，关注后该作者后续内容会得到 in-network 加权）

---

### R · 风险分 (Risk) — 权重 0.10 （但触发即重罚）

| 原生动作 | 在维度内的子权重 | 说明 |
|---|---|---|
| P(report)         | 0.40 | 举报：最严重 |
| P(block_author)   | 0.30 | 拉黑 |
| P(mute_author)    | 0.20 | 静音 |
| P(not_interested) | 0.10 | 不感兴趣 |

**为什么这个维度的权重看起来不高（0.10）但实际威胁大**：

因为它在最终公式里是 `0.10 × (100 - R)`，当 R 接近 0 时贡献接近满分；当 R 升高，不仅自己这一项掉分，还会**反映出 LLM 在评估其他维度时也会因为风险信号而下调**。所以风险维度的实际影响远超 10% 权重。

**核心可优化要素**：
- 移除推销话术
- 移除说教/居高临下感
- 移除 AI 味词汇
- 避免争议性的"三轨"话题（政治/宗教/性别）除非这是你的内容定位

**对应源码组件**：`Weighted Scorer`（负向动作有负权重）、`VFFilter`（极端情况直接过滤）

---

## 计算公式

```
Final Score = 0.35 × A + 0.25 × C + 0.20 × T + 0.10 × F + 0.10 × (100 - R)
```

各维度内部子权重用于辅助 LLM 评分时心理建模，**实际打分时不需要单独算出每个原生动作的概率**，让 LLM 直接对维度打分即可。

## 校准基准

| 总分 | 表现描述 | 占比预期 |
|---|---|---|
| 85+ | 顶尖（具备爆款潜质） | < 5% |
| 70-85 | 优秀（值得发） | ~15% |
| 50-70 | 中等（可发但可优化） | ~50% |
| 30-50 | 偏弱（建议改写） | ~25% |
| < 30 | 不建议发 | ~5% |

如果用户的推文长期分布在 65-75 区间，是正常的；如果集中在 80+，可能是 LLM 给分偏高，需要调整 Prompt 中的 calibration 提醒。
